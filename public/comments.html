Cross-Site Scripting (XSS)

XSS é uma vulnerabilidade que permite a injeção de scripts maliciosos em uma página da web, potencialmente comprometendo a segurança e a experiência do usuário. Esta demonstração mostra a diferença entre um comentário vulnerável e um comentário seguro contra XSS.

Comentário Vulnerável: Utiliza innerHTML para renderizar o conteúdo diretamente, permitindo que tags HTML, como <iframe> e <script>, sejam interpretadas pelo navegador.

Comentário Seguro: Utiliza textContent para exibir o conteúdo como texto simples, prevenindo a execução de qualquer HTML ou script injetado.

Como testar um ataque XSS:

  • Para simular o ataque, insira o seguinte código no campo de comentário vulnerável:
    • <iframe src="https://example.com" width="300" height="200"></iframe>

A demonstração mostra como ataques XSS podem afetar a página ao incluir iframes ou scripts potencialmente perigosos, destacando a importância de tratar entradas de usuário com segurança.
